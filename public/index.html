Implementation Roadmap

✓ Phase 1 — Manual Crawl (Complete)

Static HTML dashboard. Events crawled manually from 8 sources. Filter by type, month, and search. List and calendar views. Print-ready for EDDC presentations.

✓ Phase 2 — Daily Auto-Crawl (Current)

Node.js scrapers run daily at 6 AM ET via GitHub Actions. Crawl all 8 source sites, deduplicate, and write events.json. Dashboard loads dynamically from that feed. Hosted on Vercel/Netlify/GitHub Pages.

Phase 3 — Public Event Submission (Next)

Add a "Submit an Event" form (name, date, time, venue, category, description, contact). Submissions go to a moderation queue. EDDC staff approve before events go live. Could use Airtable, Supabase, or a simple API.
